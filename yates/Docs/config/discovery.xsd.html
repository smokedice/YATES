Cardinality: 0..10
Description:

Discovers tests from a database or a Python test suite. The enabled attribute disables the discovery of PythonNose tests if they are not required within a test run.

DiscoveryWorker

Cardinality: 1..1
Description:

Discovers tests by reviewing the test suite provided. The filters configuration can be used to filter the tests if not all tests are required. The enabled attribute can be used to disable the discovery of tests through the test files.

DatabaseReader

Cardinality: 1..1
Description:

Discovers tests by reading a SQLite database. These tests can be filtered by configuring the filter XML or the database can be tailored before the test execution.

Children:

Cardinality: 0..1

SourceLocation

Cardinality: 1..1
Description:

Test suite source code located. The source location is broken up into two pieces to provide the facility of selecting a certain folder within a test suite to discover tests within (basic filter).

Children:

testRoot

Cardinality: 1..1
Description:

Location where the fully test suite (utils, configuration files, etc) is located. Value can be a relative or static path e.g ../tests, ./tests, /root/tests

testPath

Description:

Folder located within the testRoot. This can be used to filter a subset of tests.
