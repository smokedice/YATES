execution

Cardinality: undefined..undefined
Description:

Most common set of parameters that could change between test suite executions

Children:

iterations

Cardinality: 1..1
Description:

Iterate the test suite. A value of 0 equates to unlimited iterations

defaultTimeout

Cardinality: 1..1
Description:

If a test does not detail a 'timeout' within it's docstring, this value is used

distributionAlgorithm

Cardinality: 1..1
Description:

How to distribute the tests between the slaves. This is multiplied by the iterations to provide the number of results expected. SingleExecution: Execute each test on a single box once ExecutionOnEachBox: Execute each test on each box ExecutionOnEachBoxEvenly: Execute each test on each box, but attempt to run a single instance of each test at a time

rebootPerTest

Cardinality: 1..1
Description:

If true, reboot the slave after each test

rebootNewBoxes

Cardinality: 1..1
Description:

If true, reboot the slave when first discovered

manualInspection

Cardinality: 1..1

stopWhenAllDead

Cardinality: 1..1
Description:

If true and all slaves are dead, stop the execution
